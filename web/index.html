What is this?

ZBOT-MOBILE is a mobile-friendly Geometry Dash macro mod. The recording / playback core is derived from FigmentBoy's zBot, and the speedhack plus several smaller behaviours are referenced from zilko's xdBot. It is built as a Geode SDK mod and compiles to a .geode file that the game loads at runtime.

This mod is a fork. The recording / playback core comes from FigmentBoy's zBot; speedhack + extras are referenced from zilko's xdBot. See CREDITS.md for the full per-file attribution and a note on the upstream license.

v1.5.9 in-game features: draggable always-visible floating ball with live state/spam dots, EclipseMenu-inspired dark/violet theme, tabbed layout (Home / Macro / Speed / Spam / Settings), saved-macros list with file size + date and an EclipseMenu-style filter, clock-style speedhack with presets, a built-in autoclicker/spammer with CPS + hold-ratio sliders, "perfect run only" recording with checkpoint-based overwrite and spam-safe spacing, persistent settings remembered between launches, and a single explicit Replay button that arms playback regardless of the Home tab radio (matcool/ReplayBot + FigmentBoy/zBot canonical playback semantics — record raw m_currentProgress, fire on frame <= currentProgress, restart cursor on every resetLevel).

This project is not a web app. There is no server or browser UI to run — the page you are viewing is just an info dashboard for the imported repository. Building and using the mod requires the Geode toolchain and a copy of Geometry Dash on Android, Windows, macOS, or iOS.

Project layout

  • src/ — C++ sources (main, gui, recording, playback, speedhack)
  • resources/ — packaged assets (e.g. logo.png)
  • CMakeLists.txt — CMake build, pulls deps via CPM
  • mod.json — Geode mod manifest
  • .github/workflows/build.yml — Android CI build

Dependencies

  • Geode SDK (via GEODE_SDK env var)
  • GDReplayFormat
  • gd-imgui-cocos
  • C++23, CMake ≥ 3.25

Build (locally, with Geode toolchain)

Requires the Geode CLI, Geode SDK, and platform-specific toolchains (Android NDK for Android targets).

# install the Geode CLI (see https://docs.geode-sdk.org)
# then, from the project root:
geode sdk install
geode build               # builds for your host platform
# or target Android explicitly:
geode build -p android64

Continuous integration

The repository ships a GitHub Actions workflow (.github/workflows/build.yml) that builds Android32 and Android64 .geode artifacts on every push to main and publishes them as a release.

Install the built mod

Drop the produced .geode file into your Geometry Dash mods folder:

Android: /storage/emulated/0/Android/media/com.geode.launcher/game/geode/mods/
Windows: %LocalAppData%\GeometryDash\geode\mods\

Recorded macros are saved under geode/macros/.
